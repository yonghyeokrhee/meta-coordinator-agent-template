# Knowledge Index

> **사용법**: 이 파일을 먼저 읽고 → 질문 유형에 맞는 파일을 순서대로 읽는다.  
> 파일을 읽을 때마다 답변이 충분한지 판단하고, 충분하면 즉시 멈춘다.

---

## 조회 순서

질문이 들어오면 아래 순서대로 파일을 확인한다.

**1단계: 01 → 02 → 03 순서로 질문 유형에 해당하는 파일을 읽는다.**

- 각 파일을 읽은 뒤 답변이 충분하면 즉시 멈춘다.
- 03은 "언제 되나요 / 알려진 이슈인가요" 유형일 때만 연다.

| 파일 | 질문 유형 |
|---|---|
| [01_system_operations_manual.md](./01_system_operations_manual.md) | 오류 / 반영 지연 / 데이터 수집 / 동기화 / 에스컬레이션 판단 |
| [02_optapex_manual.md](./02_optapex_manual.md) | 기능 정의 / 용어 / 지표 / 온보딩 FAQ / 설정 방법 |
| [03_optapex_backlog.md](./03_optapex_backlog.md) | 알려진 이슈 / 출시 예정 / 로드맵 / 가능 여부 / ETA |

**2단계: 1단계 파일로 부족할 경우에만 04 Help Center 인덱스를 탄다.**

- [04_help_center/00_Index.md](./04_help_center/00_Index.md) 를 읽고 질문과 연결된 섹션 파일 목록을 추린다.
- 추린 파일을 순서대로 읽고, 답변이 충분해지면 멈춘다.

**2단계도 부족하면**: `references/optapex-source-routing.md` 참고

## 파일 성격 요약

| 파일 | 성격 | 비고 |
|---|---|---|
| 01_system_operations_manual.md | 운영 runbook | 장애·에스컬레이션 판단용. 고객에게 내부 흐름 직접 노출 금지 |
| 02_optapex_manual.md | 기능 doc + FAQ | 기능 정의·용어·지표 의미·온보딩 FAQ |
| 03_optapex_backlog.md | 알려진 이슈·로드맵 | 상태/일정 문서. 원인 분석 문서 아님 |
| 04_help_center/ | 공개 사용 가이드 | 섹션 인덱스를 통해 접근 |

## 주의

- 이 Index는 조회 순서 결정용이다. 답변 근거는 반드시 실제 파일 본문에서 확인한다.
- 01~03으로 답변이 충분하면 04는 열지 않는다.
- 03(backlog)은 로드맵/이슈 확인 목적이 있을 때만 연다. 모든 질문에서 자동 조회하지 않는다.
