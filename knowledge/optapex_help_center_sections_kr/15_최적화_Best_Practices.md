# 15. 최적화 Best Practices

## Optimization Set 구조 설계하기

### Key Point 1 - ASIN은 언제든 이동할 수 있습니다

어떤 상품이 더 이상 해당 세트의 목표에 부합하지 않는다면, 해당 ASIN을 제거하고 다른 세트에 추가할 수 있습니다.

### Key Point 2 - 필요할 때 목표를 변경할 수 있습니다

목표를 변경하면 Optapex는 해당 목표에 맞게 최적화를 즉시 재정렬합니다.

### Key Point 3 - 모든 조합이 동일하게 작동하지는 않습니다

세트가 명확하고 일관된 목표를 가지고 있는지가 중요합니다.

### Key Point 4 - 예산은 부족하게보다 여유 있게

기본 원칙은 ASIN당 하루 최소 **$5 이상**의 예산을 확보하는 것입니다.

## 자주 발생하는 시나리오와 대응 방법

### 1. 신제품 출시 vs. 베스트셀러
- **Do:** 신제품끼리 별도의 세트를 구성하거나, 볼륨이 낮은 상품과 함께 묶어 탐색에 필요한 예산을 확보하세요.
- **Don't:** 베스트셀러와 함께 묶지 마세요.

### 2. 성장형 상품 vs. 수익형 상품
- **Do:** 성장 상품은 Maximize Sales, 마진 민감 상품은 Maximize Profit으로 세트를 분리하세요.

### 3. 시즌 상품 vs. 상시 판매 상품
- **Do:** 시즌 상품은 명확한 기간을 설정한 별도의 세트로 구성하세요.

### 4. 니치 상품 또는 저트래픽 상품
- **Do:** 비슷한 규모의 ASIN끼리 묶고, 실험 가능한 예산을 설정하세요.

### 5. 하나의 세트에 너무 많은 ASIN
- **Do:** 예산에 따라 관리 가능한 규모(예: **5~15개 ASIN**)를 유지하세요.

### 6. 모든 상품을 단일 ASIN 세트로 분리
- **Do:** 전략적으로 매우 특수하거나 예산 민감도가 높은 상품에만 단일 ASIN 세트를 사용하세요.
