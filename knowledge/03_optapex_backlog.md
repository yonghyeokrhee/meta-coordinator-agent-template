| 등급 | 처리 기한 | 기준 |
| --- | --- | --- |
| **긴급/치명적** | 즉시 파악 및 대응 | 운영 중단, 데이터 유실, 보안 침해, 광고 집행 불가, 실적에 명확한 악영향 제공, 결제 또는 회원가입 전면 장애 등 즉각 처리하지 않으면 고객 이탈과 서비스 제공 불가가 동시에 발생하는 경우 |
| **시급** | 48시간 이내 이슈 파악 및 대응 | 실적 정합성 오류, 주요 캠페인 실행·수정 불가, 백오피스 마비, 도입 문의/계약 절차 중단 등 때문에 단기 미해결 시 핵심 고객군 이탈 또는 중요한 비즈니스 피해 가능성이 뚜렷한 경우 |
| **보통** | 매주 1회 | 리포트/알림 일부 오류, 특정 계정·프로세스에서 장애, 온보딩 단계 일부 불편 등 지속 발생 시 중소 고객 이탈 가능성 또는 내부 운영상 효율 저하가 우려되는 경우 |
| **낮음** | 매주 1회 | 한정 기능의 문제나 일부 브라우저·상황에서만 노출되는 오류, 신규 기능 온보딩 내 안내 미흡 등 당장 고객 이탈이나 서비스 제공 중단 가능성이 낮은 경우 |
| **개선/참고** | 매주 1회 | 기능 개선 제안, UI/UX 피드백, 기존 프로세스 내 경미한 비효율 등 비즈니스 임팩트와 고객 이탈 가능성이 매우 낮은 경우 |

## §7. CS Agent 라우팅 힌트 + 사용 규칙

### 7.1 라우팅 힌트

- **이 문서를 우선 조회해야 하는 키워드**:
  - "언제 출시 / 언제 나와요 / 예정 / 로드맵 / 일정"
  - "알려진 이슈 / 버그인가요 / 이거 안되는데"
  - "이 기능 있나요 / 가능한가요 / 지원하나요"
  - "수정 요청 / 개선 요청 / 신규 기능"
  - 특정 페이지 키워드: "데이터 수집 100%", "Opt set 미표시", "Budget Change History", "재연동", "AUTO 상태 유지"

- **이 문서에서 답이 안 나오는 경우**:
  - 일반 응대 케이스 → [01_system_operations_manual.md](./01_system_operations_manual.md)
  - 기능 / 지표 정의 / FAQ → [02_optapex_manual.md](./02_optapex_manual.md)

### 7.2 답변 시 사용 규칙 (백로그 특화)

1. **`진행현황` 기반 표현 매핑**
   - `신규등록` / `기획·논의중` → "현재 검토 / 기획 단계이며, **구체 일정은 미정**" (시점 약속 금지)
   - `개발예정` → `Release Version` (예: "Optapex Sprint 9") 함께 안내 가능
   - `개발중` / `홀딩` → 사유 확인 후 안내. 홀딩 사유에 외부 (Stripe 기술적 제약 등) 가 있으면 그대로 인용 가능
   - `답변완료` / `개발완료` / `배포완료` → "이미 처리되었습니다" + 처리 결과 인용
2. **항상 ID 와 출처 인용**. 예: "`#125`, Optapex Backlog, 진행현황: 신규등록"
3. **민감 정보 비노출**:
   - 다른 고객사의 **Amazon Account ID**, **이메일 주소**, **member_email** 절대 노출 금지
   - 내부 employee 이름 (양도훈, 김태경, 박웅, 김나연, 이지영 등) → 외부 응답 시 "**담당 PM/책임자**" 등으로 추상화
   - 내부 swagger / Redash / DynamoDB / S3 경로 → 외부 응답에서 제거
4. **중복 ID 처리**. 같은 증상이 여러 ID 로 등록된 경우 (예: #154, #156, #150 의 "Optimization Set UI 미표시"), 가장 최근 / 가장 명확한 ID 를 대표로 인용
5. **`완료 예정일` vs `Release Version` 우선순위**. 둘 다 있으면 Release Version 이 더 신뢰. Release Version 만 있으면 정확한 일자 약속 X (Sprint 단위로만 표현)
6. **시급도(Business Impact) 노출 금지**. "긴급/치명적" 같은 내부 분류는 응답에 직접 인용하지 않음 (담당자 우선순위 신호일 뿐)
7. **검토되지 않은 항목 (진행현황 미기재)** → "최근 접수되어 검토 중" 으로 응대
