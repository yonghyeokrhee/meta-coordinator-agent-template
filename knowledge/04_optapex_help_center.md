## §1. LG Optapex 시작하기 (Getting Started)

> "Amazon 계정을 Optapex 에 연결하고 최적화 세트를 생성하는 방법을 안내합니다. LG Optapex 도입 절차는 간단합니다."

### Step 1 — Amazon 계정 연결하기
Amazon Ads Console 계정과 Amazon Seller Central 또는 Vendor Central 계정을 **모두** 연결해야 합니다.

세부 항목:
- 데이터 연동하기
- Seller/Vendor Central 계정 연결하기
- 지원 지역 및 국가 안내
- 여러 계정 추가하기

### Step 2 — 30일 Trial 활성화하기
계정 연결이 완료되면 **모든 Pro 기능이 포함된 30일 무료 체험**을 활성화할 수 있습니다.
- 광고 데이터 수집에는 **최대 72시간 소요**
- 세부: 계정 활성화하기 / 데이터 수집 범위

### Step 3 — 최적화 세트 설정하기
데이터 수집이 완료되면 최적화 세트를 설정하여 LG Optapex 의 최적화 엔진을 실행할 수 있습니다.
- 최적화 기본 개념
- 최적화 세트 생성 5단계
- Ads Console 에서 Optapex 캠페인 구조 확인하기

### Step 4 — 최적화 세트 모니터링 및 개선
적절한 시점에 최적화 세트를 조정하면 광고 성과를 크게 개선할 수 있습니다.
- 최적화 라이프사이클
- 예산 모니터링

## §2. Optapex 광고 최적화 (How Optimization Works)

### 2.1 최적화 작동 방식

Optapex 의 최적화는 **"사용자가 설정한 예산 범위 내에서 Amazon 광고 성과를 극대화하기 위해 Optapex 가 광고를 자동으로 운영하는 과정"** 입니다. 수동 관리 대신 데이터와 알고리즘을 활용하여 최적의 예산 사용 임계치를 판단합니다.

**궁극적 목표**
- 더 많은 매출과 높은 운영 효율 달성
- 반복적인 수동 업무 부담 감소

**최적화의 목적**
- 성과 우수한 상품·키워드를 우선 운영하고 저성과 영역의 불필요한 지출 감소
- 매출 변화, 경쟁 환경, Amazon 수수료 변동 등 외부 요인에 지속적 대응

### 2.2 LG Optapex: AI 와 수리최적화의 결합

Optapex 는 **"AI 와 수리최적화의 결합"** 을 핵심으로 합니다. 단순 자동화나 규칙 기반 운영과 구별되는 특징입니다. AI 를 통해 각 상품과 광고의 고유 특성과 행동 패턴을 이해한 후, 수학적 최적화 모델에 적용하여 **목표 중심의 의사결정**을 도출합니다.

| 역할 | 내용 |
| --- | --- |
| **AI** | 과거 데이터를 분석해 패턴을 식별하고, 매출·전환율·시즌성 등의 신호를 기반으로 ASIN, 키워드, 캠페인의 향후 성과를 예측 |
| **수리최적화** | 예측 결과를 실제 운영 환경에 적용. 예산 규모, 캠페인 구조, 최적화 목표와 같은 제약 조건을 고려해 최적의 예산 및 입찰 배분 방안을 계산 |

**차별점**: AI 단독으로는 예측은 가능하나 실제 예산과 입찰 배분의 정밀한 판단에 한계가 있습니다. Optapex 는 두 기술을 결합하여 단순 예측을 넘어 실행 가능한 결과를 제공합니다.

## §3. Optimization Set 관리하기

> "몇 가지 간단한 단계로 첫 번째 최적화 세트를 만들고, 더 스마트한 광고 운영을 시작하세요." 입찰가와 예산을 자동으로 처리하는 시스템.

### 3.1 왜 올바른 설정과 관리가 중요한가?

다음 네 가지 항목을 보장합니다:
- 상품이 성격에 맞는 그룹으로 정렬되었는가
- 최적화 실행에 충분한 예산이 설정되었는가
- 세트 목표와 마케팅 전략이 일치하는가
- 품절·경쟁사 활동 등 외부 요인 영향을 모니터링하고 있는가

### 3.2 기본 원칙 3가지 (Ground-Zero Rules)

| Rule | 원칙 |
| --- | --- |
| **#1** | 예산을 넉넉하게 설정하되, 시스템이 수익 체감을 감지하면 자동으로 투자 속도를 조절합니다. |
| **#2** | **2주에 한 번씩** 검토하여 예산 부족 여부를 확인하는 것이 이상적입니다. |
| **#3** | 모든 상품의 매출 확대가 필요한 것은 아니므로, 성장 곡선 끝에 있는 상품은 이익 목표로 전환 검토가 필요합니다. |

### 3.3 추가 학습 자료

- 예산 모니터링 및 제어
- Optimization Set 설정 Best Practices
- Planning with Budget Simulator
- 브랜드 보호 및 경쟁사 공략 설정

## §4. 리포트

> "한눈에 트렌드와 이상 징후를 파악하고, 바로 실행 가능한 인사이트를 확인하세요." Amazon 광고 운영 시 성과 데이터 분석의 어려움을 해결하기 위해 설계되었으며, 판매와 광고 데이터를 통합하여 시각화합니다.

### 4.1 리포트 유형 — Product 리포트 vs Brand 리포트

| 항목 | Product 리포트 | Brand 리포트 |
| --- | --- | --- |
| **데이터 범위** | Seller Central + Ad Console | Ad Console 만 |
| **광고 유형** | Sponsored Products, Sponsored Display | Sponsored Brands |
| **분석 단위** | ASIN 단위 | 캠페인 단위 |

### 4.2 Product 리포트 — 구성

| # | 구성 요소 | 내용 |
| --- | --- | --- |
| 1 | 날짜 / 필터 | 분석 범위 설정 |
| 2 | 전체 성과 (Performance Overview) | "계정 전체의 총 판매액과 광고 성과" 를 스냅샷으로 제공 |
| 3 | 트렌드 차트 | 시계열 시각화 (최대 4개 지표 권장) |
| 4 | 상품 개요 | ASIN 단위 성과 분석 |

**기능**
- 일별·주별·월별 성과 조회
- Optimization Set 별 필터링
- "재고 부족" 등 이상 징후 알림

### 4.3 Product 리포트 상세

#### 4.3.1 날짜 / 필터 살펴보기

- 트렌드 차트에서 원하는 날짜 범위 선택
- Optimization Set 기준 필터링 (복수 선택 가능)
- 재고 부족 필터링: **Days of Supply 30 / 60 / 90일 미만** ASIN 식별
- 과잉 재고 필터링: **Days of Supply 180 / 270 / 365일 초과** ASIN 식별
- 성장 / 효율 개선 가능 ASIN 필터링

> ⚠️ **데이터는 Amazon API 정책에 따라 최대 2일 지연 가능**

#### 4.3.2 데이터 비교하기

- **Compare With** 버튼 클릭으로 비교 기간 선택
- Custom period 로 시작일 지정 가능 (기간 길이는 현재와 동일)
- 비교 데이터는 점선으로 표시
- 전체 성과 기준 비교만 제공 (ASIN 단위 비교 미지원)

#### 4.3.3 어트리뷰션 이해하기

광고 클릭 후 구매 인정 기간 설정:
- 선택 옵션: **1일 / 7일 / 14일 / 30일**

**기본값** (광고 유형별):

| 광고 유형 | 기본 어트리뷰션 |
| --- | --- |
| Sponsored Products (Seller) | 7일 |
| Sponsored Display (Seller) | 14일 |
| Sponsored Products (Vendor) | 14일 |
| Sponsored Display (Vendor) | 14일 |

💡 **Tip**: 짧은 어트리뷰션과 긴 어트리뷰션 간 큰 차이는 Sponsored Display 리마케팅 고려 신호

#### 4.3.4 Product Overview 이해하기

**ASIN 상태 확인 컬럼**
- Listing Price (할인 전 가격)
- Ad Eligibility (광고 집행 가능 여부)
- BSR (Best Seller Rank)
- Fees (수수료 정보)

**이상감지 배지**

| 배지 | 의미 |
| --- | --- |
| **Low Stock** | 30일 미만 소진 |
| **Excess Stock** | 180일 이상 보유 |
| **Growth** | 높은 성장 가능성 |
| **Efficiency** | 광고 효율 개선 가능성 |

> ⚠️ **"Ineligible"** 표시는 Buy Box 상실이나 리스팅 제한을 의미.

#### 4.3.5 지표 커스터마이징

- 우측 상단 **Customization 아이콘** 클릭으로 지표 조정
- 개수 제한 없음 (가독성 유지 권장)
- 고급 지표로 수익성, 광고 효율, 참여도 분석 가능

#### 4.3.6 광고 캠페인 성과 확인

- **Product & Ad Info 토글**로 연결된 캠페인 확인
- LG Optapex 관리 여부 표시
- 캠페인 단위 정보만 제공 (광고 그룹 상세 정보 미포함)

> ⚠️ Sponsored Brand 와 Sponsored TV 는 표시되지 않음.

#### 4.3.7 마진 트렌드 이해하기 — Estimated Margin

**개념**: 총 매출에서 Amazon 수수료와 광고비를 차감한 마진

**영향 요소**
- 가격 할인
- 광고비 변동
- Amazon 정책 변경

**마진 계산에 미포함되는 항목**
- 폐기, 청산, 제거, 딜 수수료
- 세금

**수식**

```
Estimated Margin =
  (총매출 − 광고비 − 추천수수료 − FBA처리수수료 − 재고수수료
   − 보관수수료 − 패널티 − 반품/환불수수료 − 쿠폰수수료
   − 딜수수료 − 기타프로모션비용) ÷ 총매출
```

### 4.4 Brand 리포트 — 특징

- **NTB (New-to-Brand) 지표**로 신규 고객 측정

**구성 요소**

| # | 구성 요소 | 내용 |
| --- | --- | --- |
| 1 | 날짜 / 필터 | — |
| 2 | 전체 성과 (Overall Performance) | 핵심 지표 카드형 제공 |

### 4.5 Metrics Glossary (Product Report + Brand Report)

> 02_optapex_manual_onboarding.md §3 의 지표 글로사리와 일부 중복되나, 본 문서가 공식 공개판이므로 **응답 시 본 §4.5 표현 그대로 인용 가능**.

#### Product Report — Total Performance

| 지표 | 의미 | 설명 |
| --- | --- | --- |
| Total Sales | 총 매출 | 기간 내 발생한 오가닉 매출 + 광고 매출 |
| Estimated Fee | (예상) 수수료 | 기간 내 발생 매출에 대한 예상 아마존 수수료 |
| Estimated Margin | (예상) 마진 | (총 매출 − 광고 비용 − 예상 Amazon 비용) ÷ 총 매출 |
| Pageviews | 총 클릭수 | 기간 내 발생한 클릭 개수 (광고 클릭 포함) |
| Sessions | 고유 클릭수 | 기간 내 발생한 고유 클릭(방문자) 개수 |
| Buy Box % | 바이박스 획득 비율 | 바이박스 획득 시점의 조회 수를 상품 상세 페이지 전체 조회 수로 나눈 값 |
| CVR(Total) | 총 전환율 | 기간 내 구매 전환율 (광고 전환 포함) |
| Units Sold | 총 판매 수량 | 기간 내 제품 판매 개수 (오가닉 + 광고) |
| Units Returned | 환불 수량 | 기간 내 발생한 환불 제품 개수 |
| Sales Returned | 환불 금액 | 기간 내 발생한 환불된 매출액 |

#### Product Report — Sponsored Products (SP)

| 지표 | 정의 |
| --- | --- |
| SP Spend | 기간 내 발생한 총 광고 비용 |
| SP Sales | 기간 내 발생한 광고 매출 (Same SKU + Other SKU) |
| SP ROAS | SP Sales ÷ SP Spend |
| SP Units Sold | 기간 내 광고를 통해 발생한 판매 수량 |
| SP Sales (Same SKU) | 기간 내 발생한 동일 SKU 광고 매출 |
| SP Units Sold (Same SKU) | 기간 내 광고를 통해 발생한 동일 SKU 판매 수량 |
| SP ACOS | SP Spend ÷ SP Sales |
| SP Impressions | 기간 내 광고 노출 |
| SP Clicks | 기간 내 광고 클릭 횟수 |

## §5. Brand Protection Dashboard

> "Amazon 에서 지속 가능한 성장을 위한 브랜드 보호 현황을 한눈에 확인" 할 수 있도록 설계됨. 광고 집행뿐 아니라 **브랜드 키워드 검색 시 최상위 위치 확보**를 핵심 목표로 설정.

### 5.1 Overview — 3가지 핵심 영역

| # | 영역 | 내용 |
| --- | --- | --- |
| 1 | **주간 / 일간 브랜드 보호 트렌드** | 시간 흐름에 따른 브랜드 클릭 점유율 변화 추적. 광고를 통한 확보 비중과 유기적 확보 비중을 함께 표시 |

**기능**

- 고급 재고 필터 (Days of Supply 기준)

> ⚠️ 예산 페이싱은 **특정 Optimization Set 선택 시에만** 확인 가능.

- **다운그레이드 · 해지**: **다음 결제 주기 시작 시** 적용

#### 연간 플랜

- **중도 업그레이드**: 가능. 남은 기간에 대해 연간 할인율 적용한 일할 계산
- **다운그레이드 · 해지**: 미사용 개월 환불 가능하나, **연간 할인분은 환불 대상에서 제외**되며 **10% 의 조기 해지 수수료** 부과

### 8.4 주요 주의사항

- **모든 요금제는 해지 요청이 없는 한 자동 갱신**
- **월간 플랜은 환불 불가**
- 결제 미이행 시 최적화 기능 중단 가능
- 모든 결제 · 요금제 변경은 **LG CNS 검증 절차** 필요

### 8.5 FAQ 핵심

- **무료 체험 30일 후 즉시 첫 결제** (취소 미진행 시)
- 월간 → 연간 전환 가능
- 월 중간 추가 항목은 **일 단위 계산**
- ASIN 수나 매출 한도 초과 시 **커스텀 플랜 업그레이드** 안내
- 언제든지 해지 가능 (**결제 주기 종료 시 적용**)

## §9. (공지) AI Agent 사용 가이드 (Beta)

> 본 기능은 **Beta 버전** 으로 운영 중이며 지속적으로 개선될 예정.

### 9.1 핵심 기능 — 4가지

| # | 기능 | 설명 |
| --- | --- | --- |
| 1 | **광고 / 판매 지표 분석** | 계정의 주요 KPI 를 빠르게 확인하고 이상 신호를 탐지 |
| 2 | **상품 분류** | 성과 기반으로 상품(ASIN)을 그룹화하고 우선순위를 제안 |
| 3 | **실적 변화 원인 분석** | 캠페인 / 광고 / 상품 / 키워드 단위로 성과 변화의 원인을 구조적으로 설명 |
| 4 | **최적화 세트 생성 / 변경** | 분석 결과를 바탕으로 최적화 세트를 새롭게 생성하거나 수정 |

### 9.2 사용 절차 (5단계)

| Step | 설명 |
| --- | --- |
| 1-2 | **언어 설정** 후 우측 하단 플로팅 버튼 (💬) 에서 AI Agent 접속 |
| 3 | **Quick Menus** 를 참고하여 질문 입력 (Confirm, Stop, Next step 버튼 활용 가능) |
| 4 | 상품 분류 확인 → 생성 방식 선택 → 예산 설정 검토 → **최적화 SET 생성** |
| 5 | 좌측 하단 **History** 에서 이전 대화 기록 확인 |

### 9.3 주요 주의사항

- 대화 기록 다운로드 **현재 미지원**
- 리포트는 **CSV, Markdown, HTML** 형태 다운로드 가능
- 실시간 데이터 아님 (**최근 수집 데이터 기반**)
- 향후 LITE / PRO 플랜별 사용량 제한 예정
