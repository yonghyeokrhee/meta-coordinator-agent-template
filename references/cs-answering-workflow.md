# CS Answering Workflow

## 목적
Optapex CS 질문에 답할 때, 인덱스만 재사용하지 말고 실제 markdown 근거까지 직접 확인한 뒤 안전하게 답변한다.

## 답변 순서
1. 먼저 문의를 Skeleton으로 정리한다.
2. Notion `Optapex Manual > CS Intake Queue`에 접수한다.
3. `Optapex Manual`의 Knowledge Map에서 질문 축을 찾는다.
4. 연결된 실제 markdown 문서를 직접 읽는다.
5. 필요하면 backlog를 함께 확인한다.
6. 그래도 근거가 부족하면 트리아지 모드로 전환한다.

## 답변 원칙
- 사실, 운영 가이드, 추정을 구분한다.
- 인덱스만 보고 답하지 않는다.
- 고객이 바로 실행할 수 있는 단계부터 준다.
- 내부 구현 세부는 고객 답변에 불필요하면 숨긴다.
- 알려진 제한사항이면 우회 방법/기대 동작을 같이 적는다.
- 원인 미확정 상태에서는 단정하지 않는다.

## 권장 답변 포맷
- 한 줄 결론
- 고객이 해야 할 액션 1~3개
- 확인된 조건/제약
- 추가 확인이 필요한 경우 최소 질문 1개

## 답변 후 지식화
- 반복 가능성이 높은 내용이면 `CS Knowledge Sources` 또는 `CS Answer Cards` 승격을 검토한다.
- 증거 링크는 source 기준으로 남긴다.
- backlog성 문의는 backlog 문구와 상태를 우선 근거로 삼는다.

## 트리아지 전환 기준
- 고객 영향이 현재 진행 중이다.
- 계정/권한/데이터 정합성/운영 문제로 직접 조치가 필요하다.
- Knowledge Map + markdown + backlog 기준으로도 안전한 답변이 안 나온다.
- 버그 가능성이 높고 재현 또는 추가 확인이 필요하다.
