# Optapex Help Center Taxonomy

## 목적
Help Center 기반 질문을 빠르게 분류하고, 어떤 Answer Card 주제로 승격할지 정리하기 위한 보조 자료다.
이 문서는 **지식 생성/분류용**이며, 답변 절차 자체는 다루지 않는다.

## 주요 질문 축
1. 온보딩 / 계정 연동
2. Trial / 데이터 수집 대기시간
3. 최적화 세트 / 예산 운영
4. 리포트 / 지표 이해
5. 실시간 대시보드 / 데이터 차이
6. 다운로드 / 원본 리포트
7. 브랜드 보호
8. 결제 / 구독 / 플랜 변경

## 주제별 핵심 포인트
### 1) 온보딩 / 계정 연동
- Ads Console + Seller/Vendor Central 둘 다 연결 필요
- 최소 Editor 권한 필요
- 지역별 반복 연결 필요
- 신규 Ads account는 profile/활성 ASIN 조건 이슈 가능

### 2) Trial / 데이터 수집
- 무료 체험은 30일
- 초기 데이터 수집은 최대 72시간 소요 가능
- 72시간 이내는 장애로 단정하지 않음
- Brand Protection / Budget Simulator는 추가 학습 기간 필요

### 3) 최적화 세트 / 예산 운영
- Optimization Set은 생성 후 주기 점검 대상
- 예산은 지나치게 타이트하게 두지 않음
- 2주 단위 리뷰가 이상적
- 제품별 목표를 하나로 고정하지 않고 매출/이익 목적을 구분

### 4) 리포트 / 지표 이해
- Product report는 Seller Central + Ads Console 통합 관점
- SP / SD / SB 구분을 먼저 확인
- Low Stock / Excess Stock 같은 알림은 이상 징후 설명에 유용

### 5) 실시간 대시보드 / 데이터 차이
- 완전 실시간은 아님
- 일반적으로 1시간 단위 업데이트
- Reports와의 차이는 데이터 보정, 반품 반영, 집계 시점 차이를 먼저 본다
- 7일 이전 시간대별 성과는 Downloads로 유도

### 6) 다운로드
- Optapex 통합 리포트와 Amazon 원본 다운로드 모두 제공
- 원본 수치 검증 문의에 유용

### 7) 브랜드 보호
- 브랜드 키워드 클릭 점유율 / 유기 클릭 점유율을 함께 본다
- 보호 상태와 퍼널 흐름 설명에 활용

### 8) 결제 / 구독
- 월간/연간 결제 모두 가능
- 업그레이드/다운그레이드/해지 반영 시점이 다름
- 월간 플랜은 환불 불가
- 결제 실패 지속 시 최적화 중단 가능

## Answer Card 후보 체크
아래 질문이 반복되면 카드화 우선순위가 높다.
- Trial 활성화 후 데이터가 아직 안 보인다
- Reports와 Real-Time 숫자가 다르다
- Budget Simulator가 아직 안 보인다
- Brand Protection Dashboard가 바로 안 뜬다
- 여러 국가/계정을 연결할 수 있나
- 월간/연간 플랜을 어떻게 바꾸나

## 사용 규칙
- 공개 사용법/정책 질문이면 `04_optapex_help_center.md`와 함께 본다.
- 새 카드 후보를 만들 때는 이 taxonomy의 질문 축 중 어디에 속하는지 먼저 태깅한다.
